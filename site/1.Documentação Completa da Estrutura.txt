1.	Documentação Completa da Estrutura

Estrutura de Dados e Relacionamentos:
A arquitetura do seu aplicativo é baseada em uma série de entidades interconectadas, cada uma representando um conceito chave no gerenciamento de logística e transporte. Aqui está uma descrição de cada entidade, seus campos, tipos de dados, validações e como elas se relacionam:
Entidades Principais:
1.	User (Embutida no Base44):
•	Descrição: Entidade de usuário embutida do Base44, que é a base para PerfilCliente e PerfilTransportadora.
•	Campos:
•	id: string (identificador único)
•	created_date: datetime
•	full_name: string
•	email: string (único)
•	role: string (admin ou user)
•	Outros campos adicionais podem ser adicionados em entities/User.json.
•	Relacionamentos:
•	Um para um com PerfilCliente (via userIdGoogle).
•	Um para um com PerfilTransportadora (via userIdGoogle).
•	Um para um com Credencial (via userId).
2.	PerfilCliente:
•	Descrição: Detalhes específicos do perfil de um cliente.
•	Campos:
•	userIdGoogle: string (ID do usuário Google da base44) - Obrigatório
•	emailAcesso: string (Email usado para login como cliente) - Obrigatório
•	nomeCompleto: string - Obrigatório
•	cpfOuCnpj: string - Obrigatório
•	telefoneComercial: string
•	telefonePessoal: string
•	documentoCpfScan: string (URL do documento CPF ou CNPJ)
•	documentoComprovanteEndereco: string (URL do comprovante de endereço)
•	status: string (pending, approved, rejected, blocked) - Padrão: pending
•	motivoRejeicao: string
•	isPremium: boolean - Padrão: false
•	avaliacaoMedia: number (0-5) - Padrão: 0
•	totalAvaliacoes: integer - Padrão: 0
•	saldoCashback: number - Padrão: 0
•	saldoCreditos: number - Padrão: 0
•	historicoCreditos: array de objetos (tipo, valor, descrição, cotacaoId, pagamentoId, data)
•	historicoValoresAMais: array de objetos (cotacaoId, valorOriginal, valorFinal, diferenca, cashback, data)
•	historicoValoresAMenos: array de objetos (cotacaoId, valorOriginal, valorFinal, diferenca, data)
•	Relacionamentos:
•	Um para um com User (via userIdGoogle).
•	Um para muitos com Cotacao (via clienteId).
•	Um para muitos com EnderecoColeta (via clienteId).
•	Um para muitos com Produto (via clienteId).
•	Um para muitos com Destinatario (via clienteId).
•	Um para muitos com Chat (via clienteId).
•	Um para muitos com Pagamento (via clienteId).
•	Um para muitos com Avaliacao (como clienteId).
•	Um para muitos com AssinaturaPremium (via userId).

•	ncmCode: string (Código NCM do produto) - Obrigatório
